Software Overview

The Cirrus Photo is a non-contact, non-mydriatic retinal fundus camera combined with a Cirrus OCT. It supports capture modes color, G (green), R (red), B (blue) and AutoFluo (autofluorescence) as well as anterior and posterior OCT scanning capability. Cirrus Photo 800 additionally supports capture mode FA (fluorescein angiography) and optionally ICGA (indocyanine green angiography).

Images are captured at the field angles of 45 and 30 degrees.

Coaxial Overlap Check

Using the coalignment test tool, check whether the center of the fundus and the center of the OCT scan image overlap properly. This means checking that the scan, based on the fundus image, is actually positioned where it should be. Repeat the test if the system does not pass. If a test has been failed, the data recorded since the last successful test may be unreliable.

Test Tool

TeamViewer

TeamViewer is a preferred third-party remote connection application. It should be installed and licensed to each TSE on your work system.

To quickly allow remote access, guide the customer to the desktop on their instrument and have them find a browser (any browser will work though there are fewer issues with the download on Google Chrome, Mozilla Firefox, or similar.

Once in a browser, have the customer type in the following address (into the address bar, not the search box): https://tiny.cc/zeissqs. Note that the document ID ("zeissqs") is case-sensitive.

If entered correctly, this address will not necessarily load a new page, but it should trigger an automatic download of TeamViewerQS.exe, an executable quick-service client that should allow customers to provide a TeamViewer ID and a password which can be entered into the full client on the TSE's system to access the instrument.

Service Access

Fig 19. Service Access Menu

Service Access (also called Service Mode or just Service) is a set of utilities which can be used by Zeiss-trained technical support technicians to access Windows components and other useful tools.

To access Service Mode, select Tools > Service Access and at the password prompt, enter the password Meditec (case-sensitive).

NOTE! Do NOT provide this password to customers. If a customer requires access to a limited subset of Windows, they can use the access provided by selecting File > System and logging on as the Administrator account with the password 0000 (this password can be provided to customers). From the System menu, choose Network Connections which will open a shortcut to the Network Connections control panel but will use a Windows/File Explorer window to do so. From the Explorer window, type C:\ into the address field and navigate the file system from there as normal.

System menu access to the file system does not by default allow access to the hidden partitions. You can work around this by navigating to My Computer, right-clicking the icon and choosing Manage. This will open Computer Management, where you can select Disk Management, then right-click and choose Explore on the drive you wish to access.

EEPROM Tab
Fig 21. EEPROM Tab of the TestTool, highlighting the Read Parameters From buttons

Read Parameters From EEPROM

  • Displays current EEPROM values
  • Values can be entered manually
  • Values should not be "?" after reading from EEPROM

Read Parameters from File

  • Reads values from file located on the HDD
  • Can be loaded onto the EEPROM

Write Parameters to File

  • Writes current EEPROM values to file on the HDD

Write Parameters to EEPROM

  • Writes newly read or manually entered values to EERPOM

To verify EEPROM/Control Board functionality, try the following:

  1. Read parameters from EEPROM – verify that the values populate
  2. Write parameters to file – save file to desktop during this process
  3. Read parameters from file – use the parameters file you just saved
  4. Write parameters to EEPROM – verify no error in read or write

If an error shows in the information window or a read/write message is displayed during any of the steps above, suspect a bad control board. If normal function is returned during this process, there may have been a bad value stored in the EEPROM.

DICOM Gateway

Similarly to a Cirrus HD-OCT, the Cirrus Photo uses the DICOM Gateway application to configure communication with Forum or other DICOM-based systems or records software. This configuration utility allows local and remote Application Entity Titles (AETs), hostnames/IPs, and port values.

Troubleshooting

DICOM Gateway Service

DICOM Gateway is an application/utility for configuration as well as a Service running to manage DICOM communication requests. If the Service fails, DICOM connections cannot be established and errors are likely.

In the case of DICOM-related log messages or errors, check the status of the Service by accessing the Services system tool and locate the service named DICOM Gateway 2.4 (service name may be different if the operating system is earlier than Windows 7, for example Windows XP). The status should be "Running". If the status is blank or any other value, try restarting the service to restore functionality.

DICOM Gateway Failing to Launch from Cirrus Photo Application

To access the DICOM Gateway, typically use Tools > Settings > Network Tab > Select DICOM network > Click Configure DICOM gateway button

If the DICOM Gateway utility fails to launch from within the Cirrus Photo application, and the service is Running, you can attempt to launch the Gateway Configuration utility from Windows/File Explorer directly:

  • Windows 7/10 — C:\Program Files (x86)\CZM\DICOM Gateway\2.4\CZM.Networking.GatewayConfigurator.exe
  • Windows XP — C:\Program Files\CZM\DICOM Gateway\2.3\CZM.Networking.GatewayConfigurator.exe

If launched from the Explorer path, configure DICOM as you would normally and run the test. Once the tests pass, launch the Cirrus Photo application and click the Configure DICOM gateway button to verify the program starts and the tests still pass.

DICOM Gateway Files

If the DICOM Gateway Configuration utility or service are having issues, check the files in the following folders for potential issues:

  • Windows 7/10: %AppData%
  • Windows XP: C:\Documents and Settings\All Users\Application Data\Carl Zeiss Meditec\DICOM Gateway Service 2.3\

Of particular note in either location are the files GatewayRepository.xml and NIMConfig.xml.

Common Issues

"Backup of System Files Failed" Error

Verify there is enough free space on the E:\ and Z:\ partitions to rewrite the folders named SystemFilesBackup. Rename the folders in both locations (e.g. SystemFilesBackup.1).

If there is not enough disk space, delete the existing SystemFilesBackup folders.

In either case, the folders will be rewritten on the next application restart. This will also work if a specific file fails during a backup (e.g. a license file fails to backup during device start-up check).

Backup Loop Issues

If experiencing issues with a backup loop, navigate to C:\Program Files (x86)\CZM\Cirrus photo\Configuration\userconfig.xml and right-click, then choose Edit. The file will open in Notepad.

CTRL + F (or Edit > Find) to access the search box and type in "BackupRestore" (you can check the Match Case box as well).

The BackupRestore block should look something like this:

<add Key="BackupRestore" Value="" Type="System.String" Description="" Permission="0">
    <SubList>
        <add Key="LastUsedBackupFolder" Value="E:\test" Type="System.String" Description="" Permission="0" />
        <add Key="DatabaseFileToBeRestored" Value="" Type="System.String" Description="" Permission="0" />
        <add Key="IncludeExternalFilesInRestore" Value="false" Type="System.Boolean" Permission="0" />
    </SubList>
</add>

Remove anything entered between the quotation marks for the Value of key DatabaseFileToBeRestored then save the file and restart the application.
