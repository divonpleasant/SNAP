Cirrus HD-OCT Quick Reference

Models 500/5000/6000

The Cirrus HD-OCT is a non-contact, high resolution tomographic and biomicroscopic imaging device intended for in-vivo viewing, axial cross-sectional, and three-dimensional imaging of anterior and posterior ocular structures. The results are analyzed and compared to a database of age-related normal eyes to determine the variance from normal eye tissue.

The Cirrus HD-OCT is a computerized instrument that acquires and analyzes cross-sectional and three-dimensional tomograms of the eye using spectral domain optical coherence tomography (SD-OCT).

Exterior hardware components, labeled. Pictured: Cirrus HD-OCT model 500 with powered table

In low-coherence interferometry, light is sent along two optical paths, one being the sample path (into the eye) and the other the reference path of the interferometer. The light source is an 840 nm super luminescent light-emitting diode (SLD). Light returning from the sample and reference paths is combined and introduced to the detector, which is a spectrometer in SD-OCT.

Different Cirrus HD-OCT models use different technologies to create the image: All models include a CCD video camera to monitor the exterior eye and assist with scan alignment.

  • Models 5000/6000 include a Line Scanning Ophthalmo-scope (LSO)
  • Model 500 uses the OCT beam to create the retinal image

Cirrus OCT devices run on Microsoft Windows 10 (current) and Windows 7 (previous). The Windows environment is not limited, normal Windows connectivity and troubleshooting steps will be used during support interactions.

Changes to Hardware or Software

The Cirrus HD-OCT is a medical device. The software and hardware were designed in accordance with U.S., European, and other international medical device standards intended to protect clinicians, users, and patients from potential harm caused by mechanical, diagnostic, or therapeutic failures.

Customers are not meant to modify the hardware or software beyond the documented configurations listed in the user manual (IFU). Doing so can void the warranty, expose patients to harm, and force Zeiss to require an expensive recertification process to ensure the device meets all required standards.

Emphasize to customers whenever relevant that the device may have an interface similar to and behave in some cases like a commodity desktop PC, but it is not at all a standard personal computer and should never be treated as such.

Software Screens

The Cirrus HD-OCT software has five primary screen types.

The principal user interface screens of the Cirrus HD-OCT software package

1. Patient ID Screen

Search, select, or add a patient. This is the default application screen. Default tab selection is configurable by Display Options in Preferences.

2. Protocol and History Screen

This screen displays the most popular scans for the selected protocol.

3. Acquire Screen

Primary technician screen, where scans are acquired for the selected patient.

The acquire screen is comprised of four quadrants, each displaying a specialized view of the patient.

Iris Camera view, showcasing the Brightness/Contrast settings dialog
Iris Camera

The Iris Camera featured in the top left quadrant of the Acquire Screen, shows an exterior image of the patient's eye. It is used for aligning the scan with the patient's pupil in order to get a clear view into the lens to the retina. The brightness/contrast settings are of particular note because they have been the source of numerous customer calls. When these settings are adjusted too far, the Iris Camera can appear distorted and potentially unusable. The brightness/contrast settings button is sometimes referred to as the "Magic Wand" due to the original appearance of the icon:

"Magic Wand" icon

Many issues with the Iris Camera can be resolved by clicking the "Reset" button (backwards pointing arrow) which returns the brightness/contrast settings to their factory defaults.

LSLO Camera
LSLO (LSO) Camera view. Pictured (left to right): normal settings, brightness settings too low, and over-saturated settings

Also called the LSO camera, the Line-Scanning Laser Ophthalmoscope (LSLO) featured in the bottom left quadrant of the Acquire Screen, displays an image of the patient's retina as viewed through the corneal lens into the pupil.

Similarly to the Iris Camera, the "Magic Wand" or brightness/contrast settings can often be the cause for customer calls. Using the reset button or adjusting settings closer to factory default can often resolve these complaints.

OCT Scan Windows
OCT Scan showing normal scan detail OCT Scan heavily obscured with "green-yellow" static as a result of improper settings

The OCT Scan windows cover the rightmost quadrants (top and bottom) and display the OCT tomogram, or similar scan views depending on the exam protocol selected. As with the Iris Camera and LSLO Camera, the brigtness and contrast settings can have an adverse effect on the technician's ability to take a quality scan. The "Magic Wand" or brightness/contrast settings menu can help resolve these issues, particularly if returned to the factory settings.

4. Check Scan Quality Screen

Following a scan acquisition, this screen allows a technician to assess the scan's quality and determine whether to save or retake the scan.

5. Analysis and Reports Screen

From this screen, technicians, clinicians, or doctors can review, adjust, annotate, and assess scan results.

Data Management

The Cirrus HD-OCT saves data locally (to the Data disk, or E:\ drive). Zeiss strongly recommends customers archive data to a netowrk file server or a network attached storage (NAS) device, ideally with some type of data redundancy (e.g. RAID-1). It is possible to use an external hard drive for archiving data as well, but this is not recommended by Zeiss due to the relatively high failure rate of commodity external drives and configuration challenges when using review software.

Native Archive vs DICOM Archive

In addition to the Native archive, you can purchase a license for Forum that connects directly to a DICOM network which allows you to:

  • Save your data to a DICOM-compatible system automatically
  • Archive your data on a DICOM-compatible system automatically
  • Connect all Zeiss instruments to a DICOM-compatible network

IP Addresses

The Cirrus HD-OCT PC module and the microcontroller communicate via an internal network. The assigned static IP address is critical to instrument functionality and must not be changed. It is even labeled "DO NOT CHANGE". Despite this, it often gets changed by customer or IT. Always verify the internal NIC IP if the device does not pass the Start-Up Check.

The customer or their IT group should configure the External Network according to their standard addressing. When possible, encourage customers to use static IPs, particularly if they are using review station software or configuring DICOM using IP addresses to prevent DHCP lease renewals from breaking connectivity.

Troubleshooting Guide & Common Issues

Image or Scan Quality Issues

The Ocular Lens (also called the Main Lens or Patient Lens) housed in the Imaging Aperture should be cleaned regularly. The lens is easily compromised by contact with eyebrows, noses, fingers, foreheads and can become greasy or embedded with dust. If not cleaned properly according to the instructions provided in the user manual (IFU), it can streak. The lens has a protective coating which can also start to peel, crack, or flake away.

Always verify the Ocular Lens is clean! Customers will often insist the lens is cleaned regularly without verification.

Verifing Lens Cleanliness

Any image or scan quality complaints should be followed up with questions regarding the cleanliness of the lens. Don't take a customer's word for it, with the application open and on the Acquire Screen, check the Iris Camera while the customer shines a flashlight onto the lens. This can often reveal missed spots, peeling lens coating, or embedded dust, all of which can result in poor scan quality, artifacts on scans, or low signal strength issues.
