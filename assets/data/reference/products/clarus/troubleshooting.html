Clarus Troubleshooting Guide & Common Issues

Known Issues

Note: this list is transferred from Service Bulletin 038, updated May 8, 2025 (document version 16) for software version 1.1.4. Please check SIS for the most current version of this document.

# Issue Cause/Solution
124 Clarus application v1.1.4 startup stalls at MySQL bootup and triggers an emergency shutdown.

Cause: The application doesn't wait for the MySQL service to boot, but instead triggers an emergency shutdown.

Workaround: Wait a few minutes after an OS boot-up before starting the Clarus application. Clarus application v1.1.4 typically starts successfully on the second or third attempt.
123

For instruments that are synchronized, images for a selected patient are missing (showing placeholders for images) or the error message indicates:
Image(s) is missing/corrupt

Cause: A data-synchronized instrument is offline (most likely cause).

Solution:

  1. Select Settings > Data and Report > Data Synchronization
  2. Verify that synchronized instruments are online
  3. If the status is not green, ensure the devices in synchronization are online
  4. When all synchronized instruments are online, select the patient

The synchronized data and images appear normally.
122 Clarus application upgrade to v1.1.4 fails at database restore.

Cause: MySQL databases are not compatible.

Workaround: Trigger a full backup.

There are two ways to trigger a backup. Either will work for this workaround.

Method 1: Restart the Clarus Application

Method 2: Data Backup and Restore Settings

  1. Open the Clarus application
  2. Login as Administrator
  3. Select Settings > Data and Report > Backup & Restore
  4. Under Data Backup On Shutdown, click Backup Now
  5. Wait for the data backup to complete. A confirmation opens when the backup finishes
  6. Click OK
121 Double-clicking on FA stack shows only one image (instead of all images in the stack).

Cause: Double-clicking triggers the wrong mechanism.

Solution:

  1. Single-click on the FA stack; all images are selected
  2. Click Review
120 Clarus application v1.1.4 installation aborts and subsequent installation attempts fail.

Cause: MySQL 5.7 is not installed, preventing Clarus application upgrade to v1.1.4.

If the installer aborts during upgrade after the MySQL binaries are deleted, subsequent attempts to upgrade can fail.

Solution:

  1. Restore factory image
  2. Upgrade the Clarus application to v.1.1.4
119 The marked ONH (Optic Nerve Head) location is saved on the incorrect image.

Cause: Following a particular unintended workflow causes the marked ONH location to save on the incorrect image.

Unintended Workflow:

  1. During acquisition, manually mark the ONH location
  2. Immediately select a different image; the marked ONH location saves on the incorrect image

Selecting a different image before the ONH location saves completely interrupts the process.

Solution: Do not select another image immediately after marking ONH location.

Wait a moment for the ONH location save process to complete before selecting another image.

If already saved incorrectly, manually re-locate the ONH on the image and wait a moment for the process to complete.
118 Study information is disassociated from a patient.

Cause: Following a particular unintended workflow in the Clarus application causes study information to be disassociated from a patient.

Unintended Workflow:

  1. Select a patient
  2. Edit patient data
  3. Select the same patient again (without de-selecting)
  4. Edit patient data again
  5. The Patient key is set to Null in the Study table

Solution: Escalate to the L2 team for correction an include the following information:

  • A copy of the database
  • Log files
  • Audit logs
117 The interleave timer is not visible on the Clarus application Acquire screen.

Cause: Following a particular unintended workflow in the Clarus application causes the interleave timer to be no longer visible.

Unintended Workflow:

  1. Select a patient and click Acquire
    The interleave timer is visible
  2. Navigate to Settings
  3. Navigate to Acquire again
    The interleave timer is no longer visible

Solution: Navigate to the Patient screen and then the Acquire screen.
116 During acquisition, the pupil size and fixation are selectable, but unreadable.

Cause: The instrument uses multi-component names.

Multi-component banner size is larger, causing the pupil size and fixation to be unreadable.

Solution: Do not use multi-component names.

— Or —

Select the correct pupil size and fixation.

Pupil size selections: Myd is on the left. Non-Myd is on the right.

Fixation selections: Internal is on the left. External is on the right.
115 Images reported as imported successfully, but images are not present or viewable in the Clarus application.

Cause: OP images are imported without the RAW data.

The Clarus application requires RAW data for image analysis.

Solution: Import RAW data with the OP image.
114 Patient scanned images were not saved.

Cause: Following a particular unintended workflow in the Clarus application causes unsaved patient scans.

Unintended Workflow:

  1. Select a patient and go to Acquire
  2. Navigate to Settings and import DICOM files
  3. Navigate to Acquire again
    The patient was cleared from Acquire
  4. Acquire (one or more) images
    Acquired scans are not saved

Solution: Select the intended patient on the patient list and re-acquire images.

Make sure you initiate the DICOM import from the Patient screen (not Acquire or Analysis screen).
113 Clarus cannot acquire an image for a patient retrieved from the DICOM EMR/Forum/PACs MWL.

Cause: The patient retrieved from the DICOM EMR/Forum/PACs MWL does not have a last name.

The Clarus application requires a complete name to acquire images.

Solution: Add the patient's last name in the DICOM EMR/Forum/PACs system and retrieve the patient from the MWL again.
112 The Clarus application does not automatically re-open after database restoration.

Cause: Following a particular unintended workflow, the application does not restart automatically.

Unintended Workflow:

  1. Upgrade the Clarus application
  2. Open Clarus application
  3. On first boot, restore the database
    Clarus application closes and does not re-open

Solution: Restart the Clarus application manually.
111 The system crashes after database restore.

Cause: The system attempts to add an alert notice before it is ready.

Solution: Restart the Clarus application.

  3. Factory reset to create a new database

  3. Factory reset to create a new database

Cause: For Clarus application v1.1.4 or higher, if Enhanced Security is enabled, you must open the calibration wizard as administrator.

104 Mapped drives are not listed in the Clarus application.

Cause: Systems with Enhanced Security and UAC enabled do not show all mapped drives in the Clarus application.

Solution: Set the registry key to enable linked connections:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\EnableLinkedConnections = 1
103 Clarus application takes 30 minutes or more to start.

Cause: During the first launch of Clarus application v1.1.4, a database upgrade takes place.

For large databases, this upgrade can take 30 minutes or more.

Solution: Wait until the upgrade completes. This issue occurs one time (first launch).

Cause: Clarus Review application is installed in RDS mode, which disables some functions (delete patients, merge patients, re-assign exams).

Solution:

  1. Uninstall the Clarus Review application
  2. Install Clarus Review application without RDS mode
101 Image is not saved after acquisition.

Cause: The following workflow intermittently causes images to fail saving to the database:

  • Select Patient A
  • Use quick search that cannot find Patient B
    Note! If you navigate more quickly or more slowly, this issue does not occur
  • Capture an image
    Image is not saved because Patient B is not selected

Solution:

  1. Select the patient in the Patient menu
  2. Re-acquire the images
100 System does not enforce signing SMB communication.

Cause: The following local GPOs are disabled:

  • Microsoft network server: Digitally sign commuications (always)
  • Microsoft network client: Digitally sign communications (always)

Solution:

  • To disable SMB1 and enforce SMB signing run, refer to C:\Drivers\Zeiss\SecConfig\Zeiss\Disable_SMB1.CMD
    This script disables SMB1 communication which may cause communication loss with established mapped network drives. Network drives may need to be re-mapped
  • To enable SMB1 and not enforce SMB signing, run C:\Drivers\Zeiss\SecConfig\Zeiss\Enable_SMB1.CMD
099 During Zeiss Smart Services configuration, the user cannot determine what version is installed.

Cause: ZSS configuration does not identify the current installed configuration.

Solution: To determine the version of ZSS installed:

  • Select Settings > System > Maintenance
  • Scroll down to view the Zeiss Smart Service settings
  • If the setting for SSL Manager is not present, ZSS 1.0 is installed
  • If the setting for SSL Manager os present, ZSS 2.0 is installed

Note: ZSS 1.0 and ZSS 2.0 settings offer different options.
098 Operator cannot find ZSS screen recordings and screen captures saved to the local drive.

Cause: Screen recording storage locations vary depending on ZSS version.

Solution: Storage locations are as follows:

  • ZSS 1.0: C:\DCS_Write\dataaction\action\Video-RecordingFiles
  • ZSS 2.0: D:\ZSS\ScreenRecord
  • ZSS 1.0 and 2.0 screen captures: D:\ZSS\Screen-Capture
097 The Cancel button is inactive when trying to merge patients (after importing a database).

Cause: While attempting to resolve conflicting patient records (after importing database), the Cancel button looks inactive but can be clicked.

Solution: The Cancel button appears inactive, but it is active. Use the Cancel button normally as needed.

085

Failure to upgrade the Clarus application to v1.1.4:

System requires database rollback from MySQL 8.0 to Mysql 5.7 to install previous version of Clarus application.

Cause: N/A

Solution: If Clarus application v1.1.4 upgrade fails, the user can roll back to the old database.

  1. Delete the following:
    • Clarus_MySQL service
    • MySQL57
    • D:\Data
    • C:\Data
    • C:\ProgramData\MySQL Server 5/7
    • C:\ProgramData\MySQL Server 8.0
  2. Install previous version of the Clarus application
  3. Stop MySQL57
  4. Copy D:\DataBackup_Timestamp\ to D:\Data\MySQL\ and rename to folder Data
  5. Start MySQL57

083

Opening the Clarus Review application from Forum Viewer causes a failure and reports the error:
License not found

Cause: Forum Viewer version 4.4 working directory does not match the Clarus Review application.

Solution: Update the Forum Viewer file (review-sw-config.xml) WorkingDirectory field to match the Clarus Review application directory.

The default location of Forum Viewer file is: C:\Program Files\CZM\FORUM Viewer\review-config\review-sw-config-template.xml with @<hostname>\SAM\review-config\review-sw-config.xml

The default change for working directoy field: <workingDirectory>C:\Program Files\CZM\CLARUS</workingDirectory>
NOTE! This may change if Forum Viewer or Clarus Review was installed in a different location.
082 Clarus installer logs failure (Error Code 16) when installing with Federal Information Processing Standard (FIPS) enabled.

Cause: The installer uses an MD5 hash to verify that the firmware was successfully upgraded. FIPS disables the MD5 hash (considered an unsafe hash for encryption).

Start the Clarus application normally and verify it works.

If swapping hardware:

  1. Disable FIPS
  2. Uninstall and re-install the Clarus application
  3. Enable FIPS

  • Filter information is verified before presenting images in proofsheet

062 System cannot connect to an archive defined on a different instrument.

Cause: Two or more synchronized instruments use the same drive letter mapping for different locations. For example:

  • Clarus A synchronizes with Clarus B
  • Clarus A has an archive set to \\Server\Share1 as drive Z:
  • Clarus B has an archive set to \\Server\Share2 as drive Z:
  • Clarus A sees that archive data is available in both shared locations
    When Clarus A selects Archive and clicks Credential Activate, the following error message opens:

Click Dismiss.

Solution: Define unique drive letters for each shared folder.
061 Synchronization between two or more instruments fails.

Cause: Synchronization feature cannot communicate because the IP address changed.

Solution: Reboot the Clarus instrument.
059 Patient in MWL order is not associated with work order or acession #.

Cause: When you search for and select an MWL order using Advanced Search, the MWL order is not selected in the Patient window. Acquired images are not associated with the MWL order and the MWL order continues to appear in the Patient window.

Solution: Manually select the MWL order from the Patient list before starting acquistion.
058 Issuer of Patient ID is incorrectly set to CLARUS-700CL700-00000, CLARUS-500CL500-53000, or CLARUS500CL500-CL500-00000

Cause: If the Clarus application is opened prior to setting the correct serial number in the Calibration application, the Issuer of Patient ID is set to a default value.

For example, the following sequence of steps results in the Issuer of Patient ID being set incorrectly:

  1. Reimage the instrument computer
  2. Install the Clarus application
  3. Open the Clarus application
  4. Open the Calibration application

Solution: Manually modify Issuer of Patient ID to be unique.

The default naming convention for Issuer of Patient ID is the Model and Serial Number: CLARUS-XXXCLXXX-XXXXXX; for example, CLARUS-500CL500-12345
057 Issuer of Patient ID is the same on multiple instruments.

Cause: If you restore a database and select New Device, the instrument assigns the default Issuer of Patient ID.

If you restore a database and select Replacement Device, the current Issuer of Patient ID is restored to the one stored in the backup.

Solution: Manually modify Issuer of Patient ID to be unique.

The default naming convention for Issuer of Patient ID is the Model and Serial Number: CLARUS-XXXCLXXX-XXXXXX; for example, CLARUS-500CL500-12345

017 The Clarus application recieves "Critical Error; Controller communication failure"; image acquisition is disabled.

Cause: The All-in-One computer PC cannot communicate with the microcontroller over the internal network due to an IP address conflict or a configured network proxy.

IP Conflict Solution: The Clarus is not compatible with LAN or VLAN networks using IPv4 addressing in the range 192.168.111.0 to 192.168.111.254.

  1. Request that the customer update their office LAN or VLAN network to a compatible IPv4 network
  2. No changes made to the Clarus instrument can resolve this issue
  3. Caution! Do not change the IPv4 address of the internal network

Network Proxy Configuration Solution: Diable network proxy on the Clarus instrument.
015 ONH animation / calculation did not save.

012 The Clarus application fails to launch.

Cause: Multiple files with manufacturing.settings.xml in the name exist in the folder C:\Program Files\CZM\CLARUS\Configuration

For example:

  • manufacturing.settings.xml
  • backup.manufacturing.settings.xml

Solution:

  • Ensure there is only one file with manufacturing.settings.xml in the name in the folder C:\Program Files\CZM\CLARUS\Configuration
  • Delete all additional files with manufacturing.settings.xml in the name from the Configuration folder
  • Do not save a backup file for manufacturing settings in the Configuration folder
