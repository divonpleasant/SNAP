If the database start-up check appears to hang at the 20% mark or there are errors related to connection to database, failure to locate host machine, or an undefined service ("czm_db"), most likely the issue is the review station cannot connect to the instrument's database instance.

To resolve the issue:

  • Note the IP address and host name of the target Cirrus HD-OCT system
  • Verify (using UNC or ping) that the review station can connect to the instrument, specifically the path E:\SDOCT\DataBase\
  • Make note if the customer has created a network share for the database location; often you will see a mapped drive letter (e.g. F:\ or Z:\) which points to a share named "Database Instrument"
  • Ensure the review station software is configured to run in "Native" or archive mode (check in C:\Program Files\CZM\Cirrus HD-OCT\Bin\SdoctApp.exe.config and check the line >add key="ArchiveMode" value="Native"<; if the value is DICOM, edit the file with Administrator permissions and restart the application)
  • When the review station application starts, it should pause when the database start-up check is at 20% and open a Finder window requesting a path to the database index file, which is always located on the instrument itself unless in a Forum environment (i.e. running in mode DICOM)
    • Use either the network share path if configured (e.g. F:\Database Instrument\ZDB.ib) or a UNC path (e.g. \\192.168.1.10\SDOCT\DataBase\ZDB.ib) to locate the index file
    • NOTE: there are pros and cons to using an IP address vs a hostname in a UNC address:
      • IP addresses are prone to issues in DHCP environments in the case of IP reassignment
      • Hostnames are prone to DNS-related issues
      • If static IP assignment is possible, encourage customers to configure their instruments with a static IP and use IP address UNCs for review station configs
      • Otherwise, use whichever option is easiest for the local IT group to troubleshoot
    • If credentials are required to access the remote index file, be sure to check the "Remember Credentials" box to avoid having to re-login
    • In some cases there may be more complex communication issues between the review station and the instrument
      • Sometimes these can be resolved by enabling Network Discovery on the OCT instrument
      • Turning on (enabling and ensuring the status RUNNING) the service(s) "Function Discovery Resource Publication" and/or "Function Discovery Provider Host" on the OCT instrument can also help
      • Verify the network permissions are configured for authenticated Zeiss users on the folders SDOCT and DataBase on the OCT instrument
      • Ensure the environment does not use Forum (if so, the software mode should not be Native, rather it should be Dicom and the database instance would be local to the Review Station)
      • Ultimately this communication is the responsibility of the customer's IT group
