Notes for Auditors:

  • Each discrete data point can be marked in one of two ways: flagged or comment
  • Flag data points that are wholly inaccurate (e.g. "supported: true" when it should be "supported: false")
  • Comment data points that require additional context such as typos ("short_name: Cirris HD-OCT" instead of "short_name: Cirrus HD-OCT"), updates ("eos_date was recently set for this coming October"), or questions ("Is this phone number format acceptable?")
  • Empty data values are not considered incorrect ('flagged') unless they are top-level or identifying (i.e. a product's model name or a user's last name); use 'comment' instead to provide the expected value
